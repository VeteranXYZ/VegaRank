# Chinese Terminology System

## Purpose

本文档定义 scanner codebook 的中文研究语言标准。中文词典不是英文词典的逐字翻译，而是面向产品使用场景的专业中文术语层，用来解释扫描代码、研究优先级、风险约束、证据可靠性和人工复核边界。

中文表达应服务于研究工作流：帮助用户理解当前证据、风险和复核顺序，而不是发出交易指令或承诺结果。

## Product Boundary

trade-scanner 是专业 crypto 技术研究系统。它用于排序技术研究候选、解释扫描证据、辅助人工复核。它不提供投资建议，不发出交易指令，也不预测未来价格。

中文文案必须保持以下边界：

- 说“观察、跟踪、复核、等待、降低优先级、排除”。
- 不说“买、卖、做多、做空、进场、出场、目标价、止损位”。
- 不把研究优先级写成交易建议。
- 不把证据置信度写成上涨把握、成功率或胜率。

## Chinese Writing Principles

中文表达应专业但可读。句子要短，语义要直接，避免券商研报腔、学术腔和机械翻译腔。

写作原则：

- 克制、稳定、研究导向。
- 不情绪化，不营销化，不使用币圈黑话。
- 不喊单，不承诺结果。
- 不泄露公式、阈值、权重或后端实现细节。
- 优先使用自然中文，而不是照搬英文句式。
- 对风险保持明确，对证据不足保持谨慎。

不推荐：

“该资产当前呈现多因子风险调整后研究优先级边际改善。”

推荐：

“结构有所改善，但风险约束仍需复核。”

## Domain-Specific Terminology

“置信度”只表示证据可靠性。可以写“证据置信度”“证据可靠性”“判断基础更充分”“当前证据更完整”。不要写“上涨把握”“成功率高”“胜率高”“确定性强”或“大概率上涨”。

“研究合格”用于 Eligible，表示结构证据和风险条件足以保留在研究范围内。“合格”只能作为极短徽章使用，不能在完整标签或解释中单独替代“研究合格”。

“高优先级复核”用于 High Priority。它表示更靠前的人工复核顺序，不表示更高胜率、更高确定性或交易建议。

“避免追高”是风险提示，不是交易指令。它表示价格或动量乖离偏高，后续反应的风险回报比不佳。

“波动率压缩”不是自动利好。只有在趋势、结构和流动性共同支持时，压缩状态才更接近建设性背景。

“流动性约束”表示交易活跃度或可靠性不足，会降低证据可靠性和研究优先级。

## Explanation Templates

建设性但未确认：
“结构正在改善，但确认仍有限。该结果更适合继续观察，而不是直接提升到高优先级复核。”

建设性且风险可接受：
“该结构具有建设性证据，且风险约束仍处于可接受范围，可保留在研究队列中。”

高优先级：
“该结果同时具备较强结构质量、较好的证据可靠性和可接受的风险约束，适合高优先级人工复核。”

过热或追高风险：
“动量或价格乖离偏高，追高可能带来较差的风险回报比，因此应降低研究优先级。”

低置信度：
“当前可靠证据有限。在出现更多历史样本或确认之前，应谨慎看待该结果。”

流动性受限：
“交易活跃度低于偏好的研究范围。流动性约束降低了该结构的证据置信度。”

波动率压缩：
“波动率处于压缩状态；只有在趋势、结构和流动性共同支持时，它才更接近建设性背景。”

延续性不足：
“近期延续性较弱或不稳定。该结构需要更清晰的确认，才能获得更高优先级。”

风险调整后优先级：
“结构质量具备一定建设性，但风险约束降低了最终研究优先级。”

数据质量排除：
“由于数据质量或历史样本深度不足，该结果被排除出优先研究范围。”

## Disclosure Boundary

中文词典可以解释代码含义、研究相关性、证据约束和人工复核边界。不得披露：

- 私有公式
- 精确阈值
- 私有权重
- 后端实现细节
- 原始指标投票
- 投资建议
- 交易指令
- 未来价格预测

不要写：

- “RSI 大于 70 时触发”
- “volumeRank 小于 25 时触发”
- “riskPenalty 乘以 0.45”
- “公式计算得出”
- “阈值等于”
- “私有权重”

应写：

- “动量偏高”
- “流动性低于偏好的研究范围”
- “风险约束降低研究优先级”
- “证据可靠性有限”

## Relationship to Phase 20 English Codebook

Phase 20 英文 codebook 是语义来源。中文词典应保持同一研究边界和代码含义，但不做机械翻译。中文表达可以更短、更自然，只要不改变代码语义。

本阶段不改变英文语义、不改变 API shape、不改变评分逻辑、不改变数据库结构。

## Future UI Shell and Button Naming Phase

本阶段只处理 scanner codebook 中文术语。页面标题、导航、按钮、筛选器、表格列名和其他 UI shell 文案可在后续阶段统一处理。

后续 UI 命名阶段应复用本文件术语，尤其是“研究优先级”“人工复核”“高优先级复核”“研究合格”“证据可靠性”“流动性约束”等边界词。
